# 영상 촬영용 데모 데이터

홍보·사용설명 영상 만들 때 화면에 채워질 더미 데이터 시드.
**로컬 dev 환경 전용** — 운영 DB 에는 절대 실행하지 마세요.

## 한 번에 준비하기

```bash
# 1. 로컬 dev 환경에서 (운영 환경변수 unset 후)
unset DATABASE_URL PRISMA_SCHEMA

# 2. 시드 두 개 모두 실행
npm run seed:demo          # DB 채우기 (아동 12명 + 일정표/기록지 3개월치)
npm run seed:demo-excel    # 엑셀 3개 생성 (samples/demo/)

# 3. 개발 서버 시작
npm run dev

# 4. 브라우저
# http://localhost:3000
# 로그인: demo@baroilji.com / demo1234
```

## 생성되는 것

### DB (`npm run seed:demo`)
- 센터: **꿈나라발달언어센터**
- 사용자: **김다온** (언어재활사, OWNER, demo@baroilji.com / demo1234)
- 아동 12명 (가공 이름): 김가나, 이다라, 박마바, 최사아, 정자차, 강카타, 윤파하, 임가다, 송라마, 조바사, 한아자, 백차카
- 일정표·기록지:
  - **지난-지난달**: 12명 일정표 + 12명 기록지 (전월 비교 시연용)
  - **지난달**: 12명 일정표 + 12명 기록지 (한꺼번에 다운로드 시연용)
  - **이번달**: 12명 일정표 + **3명만** 기록지 (대시보드 "미작성 9건" 시연용)

> 매 실행 시 기존 demo 계정 통째로 지우고 재생성 → 안심하고 반복 가능.

### 엑셀 (`npm run seed:demo-excel`)
`samples/demo/` 폴더에 3개 파일:
- **서비스제공내역_정상.xlsx** — 깔끔한 케이스, 기록지 자동완성 시연용
- **서비스제공내역_위반.xlsx** — 결제 간격 위반 2건 포함, 승인내역 점검 시연용
- **서비스제공내역_소급.xlsx** — 소급결제 1건 포함, 소급결제 알림 시연용

## 영상별 활용 가이드

| 영상 | 추천 화면 흐름 | 사용할 데이터 |
|---|---|---|
| 쇼츠 1 (일정표) | /schedule → 김가나 선택 → 패턴 → 자동 생성 → 한글파일 | 시드만 |
| 쇼츠 2 (기록지 자동완성) | /record → 정상 엑셀 드래그 → 김가나 탭 → 한글파일 | 시드 + 정상 엑셀 |
| 쇼츠 3 (승인내역 점검) | /approval-check → 위반 엑셀 드래그 → 빨간 행 강조 | 위반 엑셀 |
| 쇼츠 4 (한꺼번에 다운로드) | /export → 일정표 → 지난달 → 전체 선택 → 다운 | 시드만 |
| 풀영상 2:30 | 대시보드 → 일정표 → 기록지 → 점검 → 한꺼번에 → 도움말 | 전부 |

## 영상 촬영 후 정리

```bash
# DB 의 demo 계정·데이터만 통째로 지우려면 재시드(=삭제+생성) 한 번 더:
npm run seed:demo

# 완전 초기화하려면 sqlite 파일 삭제:
rm prisma/dev.db
PRISMA_SCHEMA=sqlite npx prisma migrate dev
```

## 주의

- 모든 이름·전화·주소는 **가공** 입니다. 실제 환자·기관 정보가 절대 들어가지 않게 작성됐어요.
- 운영(baroilji.com) DB 에는 절대 실행하지 마세요 — 시드 스크립트가 `demo@baroilji.com` 계정을 통째로 삭제·재생성합니다.
