# 시작 가이드 — Claude Code로 우리 센터 프로그램 만들기

> 코딩 경험이 거의 없으셔도 따라올 수 있게 만들었어요.
> **원장님은 코드를 쓰지 않습니다.** Claude Code에게 "이렇게 만들어줘"라고 말하고, 결과를 확인하고, 피드백을 주는 역할입니다.

---

## 마음가짐 먼저

- 한 번에 다 만들지 않습니다. **작은 조각으로 나눠** 하나씩 완성합니다.
- 막히면 Claude Code에게 그냥 한국어로 "이게 안 돼", "이 부분 이렇게 바꿔줘"라고 말하면 됩니다.
- 시간이 걸립니다. 조급해하지 마세요. 며칠~몇 주에 걸쳐 조금씩 쌓아가는 일입니다.
- 이 채팅(지금 저)은 **기획·설계 파트너**, Claude Code는 **실제로 만드는 일꾼**입니다. 막히면 다시 여기로 와서 물어보세요.

---

## 0단계 — 준비물

1. **컴퓨터** (Windows 또는 Mac). 두 OS 다 됩니다.
2. **Claude 구독** (Pro 이상). Claude Code를 쓰려면 필요합니다.
3. 시간 여유가 있는 날 1~2시간. 첫 설치만 차분히 하면 그 다음은 쉽습니다.

---

## 1단계 — Claude Code 설치 (가장 쉬운 방법)

코딩이 처음이시면 **터미널 없이 쓸 수 있는 방법**을 추천합니다.

### 방법 A — Claude 데스크톱 앱 (터미널 불필요, 초보 추천)
- Claude **데스크톱 앱**에는 Claude Code가 내장되어 있습니다. 터미널이나 Node.js를 따로 설치할 필요가 없어요.
- 데스크톱 앱을 설치하고 로그인하면, 폴더를 열어 바로 작업을 시킬 수 있습니다.

### 방법 B — 네이티브 설치 (터미널 사용, 2026년 공식 권장)
- 터미널에 익숙해지고 싶다면 네이티브 설치 프로그램을 씁니다. 의존성(Node.js 등) 없이 설치되고 자동 업데이트됩니다.
- Windows는 추가로 "Git for Windows"가 필요할 수 있습니다.
- 정확한 최신 설치 명령은 공식 문서를 보세요: https://docs.claude.com/en/docs/claude-code/overview

> 둘 중 **방법 A(데스크톱 앱)**로 시작하시길 권합니다. 가장 막힘이 적어요.

---

## 2단계 — 프로젝트 폴더 만들기

1. 컴퓨터에 폴더를 하나 만듭니다. 예: `바탕화면/ondam-app`
2. 그 안에 `samples` 폴더를 만들고, 아래 파일들을 넣습니다.
   - 일정표 양식 .hwp
   - 기록지 양식 .hwp
   - 전자바우처 서비스제공내역 .xls (샘플 한 달치)
   - 우리가 만든 `prototype.html`
3. 폴더 맨 위에 `CLAUDE.md` 파일(제가 같이 드린 명세서)을 넣습니다.
   - Claude Code는 이 파일을 자동으로 읽어 프로젝트를 이해합니다.

---

## 3단계 — Claude Code 열고 첫 지시 내리기

Claude Code에서 위 폴더를 엽니다. 그리고 아래 문장을 **그대로 복사해서** 붙여넣으세요.

```
이 폴더의 CLAUDE.md를 읽고 프로젝트 전체를 파악해줘.
그리고 samples 폴더의 .hwp 양식 파일들과 .xls 엑셀 파일,
prototype.html을 모두 열어서 구조를 정확히 확인해줘.

나는 코딩을 거의 모르는 비개발자야. 그러니까:
- 한 번에 다 만들지 말고, 작은 단계로 나눠서 진행해줘.
- 각 단계마다 "지금 무엇을 만들었고, 어떻게 확인하면 되는지"를 쉬운 말로 설명해줘.
- 내가 직접 입력해야 하는 명령어가 있으면 정확히 알려줘.

먼저 전체 작업을 어떤 단계로 나눌지 계획만 세워서 보여줘.
아직 코드는 짜지 마.
```

Claude Code가 계획을 제시하면, 마음에 들 때까지 "이건 빼고", "이 순서로 해줘" 하고 조율하세요. 그 다음부터 한 단계씩 진행합니다.

---

## 4단계 — 권장 진행 순서 (단계별)

아래 순서대로 Claude Code에게 하나씩 시키면 됩니다. **각 단계가 끝나면 직접 눌러보고 확인**한 뒤 다음으로 넘어가세요.

**1단계: 프로토타입을 진짜 웹앱으로**
- prototype.html을 기반으로 실제로 돌아가는 웹앱의 뼈대를 만든다.
- 일정표 생성기 + 기록지 자동완성(엑셀 업로드) 화면이 브라우저에서 동작.

**2단계: 아동·치료사 정보 저장**
- 아동을 한 번 등록하면 매월 재입력 없이 불러오게 한다 (SQLite 저장).
- 치료사 목록 관리.

**3단계: 일정표 한글파일(.hwp) 출력**
- 기존 일정표 양식 그대로 .hwp(또는 .hwpx)로 다운로드. (어려우면 PDF 먼저)
- 공휴일 자동 표기 연결.

**4단계: 기록지 한글파일(.hwp) 출력**
- 엑셀 업로드 → 아동별 기록지 .hwp 일괄 다운로드.

**5단계: 치료사 로그인**
- 치료사별 로그인. 본인 담당 아동만 보이게.
- 원장 관리자 화면(전체 현황·미작성 회기).

**6단계(선택): 인터넷 배포**
- 센터 PC 로컬 실행으로 충분하면 여기서 멈춰도 됩니다.
- 여러 PC에서 쓰려면 국내 클라우드 배포를 Claude Code와 진행.

---

## 5단계 — 막힐 때 쓰는 말들

Claude Code에게 그냥 한국어로 말하면 됩니다. 예:
- "방금 만든 화면이 안 열려. 어떻게 확인해?"
- "이 버튼을 눌렀더니 에러가 나. 화면 그대로 붙여넣을게: (에러 복사)"
- "이 부분은 우리 양식이랑 달라. 이렇게 바꿔줘: (설명)"
- "지금까지 한 걸 저장(백업)하려면 어떻게 해?"

---

## 가장 어려운 부분 미리 귀띔

**한글파일(.hwp) 자동 생성**이 이 프로젝트에서 제일 까다롭습니다.
- 막히면, 우선 **PDF로 출력**되게 해서 전체 흐름을 먼저 완성하세요.
- 그 다음에 .hwp 생성을 따로 붙이면 마음이 편합니다.
- 정 안 되면 다시 이 채팅으로 와서 "한글파일 생성이 막혔어"라고 하시면 방법을 같이 찾아드릴게요.
