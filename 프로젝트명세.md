# 온담말언어발달센터 — 발달재활 통합관리 프로그램 (MVP 명세서)

> 이 문서는 Claude Code가 프로젝트의 목적과 규칙을 이해하기 위한 명세서입니다.
> 프로젝트 폴더 맨 위에 `CLAUDE.md`라는 이름으로 두면 Claude Code가 자동으로 읽습니다.

---

## 1. 한 줄 요약

언어치료센터(온담말언어발달센터)의 **발달재활서비스 일정표·기록지 작성을 자동화**하는 웹 프로그램.
치료사들이 매월 반복하던 수작업(양식 복붙, 날짜·시간·승인번호 입력)을 없애고, **치료 내용 작성에만 집중**하게 한다.

이번 단계는 **우리 센터 한 곳만 쓰는 MVP**다. (다른 센터 판매·멀티테넌트는 나중 단계)

---

## 2. 사용자

- **치료사 (약 10명)**: 본인이 담당하는 아동의 일정표를 만들고, 회기 기록을 작성한다. 본인 담당 아동만 보인다.
- **원장(관리자)**: 전체 치료사·아동 현황을 본다. 미작성 회기를 모니터링한다.

---

## 3. 핵심 기능 두 가지

### 기능 A — 일정표 생성 (월초)

다음 달 서비스 일정표(.hwp)를 만든다.

**입력**
- 아동 성명, 담당 치료사, 서비스 종류(언어재활/놀이치료/감각통합치료 등, **자유 입력 가능**)
- 대상 월
- 반복 요일 패턴 (예: 수·목)
- 목표 회기 수 (4~8회, 센터·아동마다 다름)
- 치료 시간대

**자동 처리 규칙 (중요)**
1. 다음 달 달력을 자동으로 그린다.
2. **한국 공휴일을 자동 표기**한다 (공공데이터포털 "특일 정보" API 연동 권장. 안 되면 우선 하드코딩).
3. 반복 요일에 회기를 자동 배치하되, **공휴일에 걸린 회기는 제외**한다.
4. **목표 회기 수를 채우는 것이 최우선.** 공휴일·연휴로 빠진 회기는 다른 날로 보충해야 한다.
5. **보충(보강) 날짜는 시스템이 자동으로 정하지 않는다.** 치료사가 어머님과 통화 후 직접 지정한다.
   시스템은 "목표 5회 / 현재 4회 (1회 부족)" 같은 **카운터로 부족분만 알려준다.**
6. 회기마다 **시간을 개별로 변경**할 수 있어야 한다 (보강은 다른 시간에 잡히는 경우가 많음).
7. **작성일자 = 대상 월 전달의 말일** (자동 계산).

**치료 시간대 (11개 고정 슬롯, 각 50분)**
```
09:00~09:50, 09:50~10:40, 10:40~11:30, 11:30~12:20,
(점심)
13:30~14:20, 14:20~15:10, 15:10~16:00, 16:00~16:50,
16:50~17:40, 17:40~18:30, 18:30~19:20
```

**일정표 양식 구조 (출력물은 이 순서·레이아웃 그대로)**
- 제목: `서비스 일정표 (N월)`
- 상단 메타표:
  - `사회복지서비스 관리번호` (입력) / `성 명`
  - `사회복지서비스 제공자` (치료사) / `작성일자` (전달 말일 자동)
- 달력 (일~토). 회기 있는 칸에 **시간 범위 전체** 표시 (예: `16:00~16:50`, 한 칸 한 줄 아님—두 줄로 줄바꿈 허용). 공휴일 칸엔 공휴일명.
- `서비스 제공현황` 표: 서비스 제공자명 | 전화 | 담당(**수정 가능**) | 서비스 종류(**수정 가능**) | 주기(요일, 자동) | 제공일(날짜들, 자동)
- `서비스 비용` 표: 서비스 종류 | 서비스 단가(/회)(**수정 가능**) | 횟수(회기 수 자동) | 총 서비스 가격(단가×횟수 **자동 계산**) | 본인부담금(**수정 가능**)

**기본값 (온담말 기준)**: 제공자명 `온담말언어발달센터`, 전화 `775-0047`, 단가 `65,000원`, 본인부담금 `0`.

### 기능 B — 기록지 자동완성 (월말)

전자바우처 결제내역 엑셀을 올리면, 아동별 발달재활서비스 제공 기록지(.hwp)를 자동으로 채운다.

**입력**
- 전자바우처(socialservice.or.kr)에서 치료사가 다운로드한 `서비스제공내역.xls` 파일.
- 이 엑셀은 **치료사 본인 것**이며, 안에 담당 아동 여러 명이 섞여 있다.

**엑셀 컬럼 (헤더 행 기준, 실제 파일에서 확인됨)**
```
번호, 시/군/구, 대상자, 생년월일, 대상자 전화번호, 대상자 휴대폰번호,
사업구분명, 사업유형명, 서비스유형명, 제공기관명, 결제일자, 결제시간,
서비스이용일자, 서비스시작시간, 서비스종료시간, 결제수단, 결제금액,
결제구분, 제공인력인원수, 승인번호, ...
```
- 엑셀 맨 윗줄에 `제공인력 이름`(치료사명)이 있다.
- 헤더 행은 `대상자`와 `승인번호`가 동시에 들어있는 행으로 찾는다.
- 날짜는 `2026.04.22` 형식 문자열. 시간은 `16:48` 형식.

**자동 처리 규칙 (중요)**
1. `대상자` 이름으로 행을 그룹핑 → 아동별 기록지로 분리.
2. 각 아동 행을 `서비스이용일자` 오름차순 정렬.
3. 기록지 상단표 채우기:
   - 월·일: `서비스이용일자` → `4/22` 형식
   - 종료시간: `서비스종료시간` 그대로
   - **시작시간 = 종료시간 − 50분** (자동 계산, 치료사가 필요시 수정 가능)
   - 바우처(분)/추가구매(분): **아동별로 처음 설정한 값을 그대로** 사용 (수정 가능, 기본 40/10)
   - 총 이용금액: 수정 가능 (기본 65,000)
4. 기록지 하단 "상태 및 결과 기록" 표 채우기:
   - 서비스 제공일자 = `서비스이용일자`의 '일'
   - 승인일자 = `결제일자`의 '일'
   - 승인번호 = `승인번호` 그대로
5. **검토: `서비스이용일자` ≠ `결제일자`(=승인일자)이면 해당 회기를 색으로 강조 + 경고 표시.** (보통 같지만 어긋나는 경우를 잡는 안전장치)

**치료사가 직접 작성하는 칸 (자동 채우지 않음)**
- 회기별 `이용자의 상태 및 서비스 결과`
- `부모 상담 종합 의견`

**기록지 양식 구조 (출력물은 이 순서·레이아웃 그대로)**
- 제목: `발달재활서비스 제공 기록지 (N월)`
- 메타표: 제공기관명 / 이용자(성명·생년월일) / 관리자 서명 / 보호자 서명
- 상단표(회기별 열): 월·일 | 시작시간 | 종료시간 | 바우처(분) | 추가구매(분) | 총 이용금액 | 이용자(확인)
- 하단표(회기별 행): 서비스 제공일자 | 승인일자 | 승인번호 | 이용자의 상태 및 서비스 결과
- 부모 상담 종합 의견란

## 4. 데이터 모델 (MVP, 단순하게)

- **치료사 Therapist**: id, 이름, 로그인 정보
- **아동 Child**: id, 성명, 생년월일, 담당 치료사, 서비스 종류, 바우처분/추가구매분 기본값, 단가, 본인부담금, 반복요일 패턴, 목표 회기 수, 기본 시간대
- **월 일정표 Schedule**: 아동, 대상 월, 회기 목록[{날짜, 시간, 보강여부}], 관리번호
- **회기 기록 SessionRecord**: 아동, 날짜, 시작/종료시간, 승인번호, 승인일자, 바우처/추가구매분, 금액, 상태·결과 텍스트
- **월 기록지 MonthlyRecord**: 아동, 대상 월, 회기 기록 목록, 부모상담 종합의견

> 핵심: 아동 정보를 한 번 등록해두면, 매월 일정표·기록지에서 재입력 없이 불러온다.

## 5. 기술 스택 (Claude Code가 확정하되, 권장안)

- **프레임워크**: Next.js (프론트+백엔드 한 프로젝트로) — 초보자에게 관리가 가장 단순.
- **데이터 저장**: SQLite (파일 하나, 설치 불필요) — 한 센터 MVP에 충분. Prisma ORM 사용 권장.
- **한글파일 생성**: HWPX 템플릿 치환 방식.
  - 센터의 기존 .hwp 양식을 .hwpx로 저장 → 빈칸에 `{{성명}}`, `{{승인번호}}` 같은 자리표시자 삽입.
  - 생성 시 .hwpx(zip) 풀어서 XML의 자리표시자만 데이터로 치환 → 다시 압축 → .hwp/.hwpx로 다운로드.
  - (대안: 우선 화면/PDF로 출력하고, .hwp 생성은 다음 단계로 미뤄도 됨)
- **엑셀 읽기**: SheetJS(xlsx) 라이브러리. .xls(구형 BIFF)도 읽음.
- **공휴일**: 공공데이터포털 "특일 정보" API. 연동 전엔 2026년 공휴일 하드코딩.
- **실행/배포**: MVP는 **센터 PC에서 로컬 실행**으로 시작해도 충분(개인정보가 외부로 안 나가는 장점). 안정화 후 국내 클라우드 배포 고려.

## 6. 개인정보 보호 (MVP에서도 반드시)

아동 정보·승인번호·보호자 연락처를 다룬다. 최소한:
- 치료사 로그인(비밀번호) 필수. 본인 담당 아동만 접근.
- 로컬 실행 시 DB 파일 백업 정책 마련.
- 클라우드로 갈 경우 국내 리전·HTTPS·저장 암호화 필수.
