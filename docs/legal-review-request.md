# 바로일지 — 법률 검토 요청서

> 본 문서는 변호사 검토를 위한 요청서입니다. 뒤에 [첨부 A] 이용약관, [첨부 B] 개인정보 처리방침, [첨부 C] 치료사용 처리방침 템플릿, [첨부 D] 회원가입 동의 문구가 이어집니다.

## 1. 서비스 개요

- **바로일지**는 발달재활서비스(발달바우처) 제공인력(치료사)이 매월 작성하는 **서비스 일정표·제공기록지·제공내역 점검**을 자동화하는 1인용 웹 SaaS입니다.
- 전자바우처 결제·정산 시스템이 **아니며**, 결제·청구·정산과 연계하지 않는 **행정 보조 도구**입니다. 생성물(일정표·기록지·점검 결과)은 참고용이며 최종 책임은 이용자에게 있습니다.
- 취급 데이터: 아동 성명·생년월일·관리번호, 회기 일정, 제공기록, 본인부담금 등. **아동의 장애·발달 관련 정보(민감정보)** 를 포함할 수 있습니다. (주민등록번호 등 고유식별정보는 수집하지 않음)

## 2. 전제한 법적 구조 (검토 대상)

1. **개인정보 처리 지위**: 아동·보호자 정보에 대하여 **이용자(치료사)가 개인정보처리자**, **회사(바로일지)가 수탁자(처리자)** 라는 구조(개인정보 보호법 제26조 업무위탁)로 설계했습니다.
2. **위탁과 동의**: 위탁은 제3자 제공이 아니므로 **보호자(정보주체)의 별도 동의 없이** 가능하며, ① 위탁 사실을 **이용자 본인의 처리방침에 공개**(첨부 C 템플릿 제공), ② **위탁계약**(본 약관·처리방침이 그 효력), ③ **안전성 확보조치**로 충족된다고 보았습니다.
3. **정부 표준 서류 미수정**: 전자바우처 표준 동의서·서류는 이용자가 임의 수정할 수 없으므로, 위탁 공개는 정부 서류가 아닌 **이용자 본인 처리방침**으로 처리하도록 설계했습니다.
4. **회원(치료사) 본인 정보**: 회사가 개인정보처리자로서 가입·인증·서비스 제공을 위해 처리.
5. **데이터 소유·격리**: 이용자 데이터의 권리는 이용자에게 있고, 1인 사물함으로 격리 보관. 외부(전자바우처 등) 전송 없음.

## 3. 검토 요청사항 (질문)

1. **위탁 구조의 타당성**: 아동 민감정보를 다루는 본 서비스에서 ‘이용자=처리자 / 회사=수탁자’ 위탁 구조와, **보호자 별도 동의 없이 처리방침 공개로 충족**된다는 해석이 적정한지.
2. **민감정보·고유식별정보**: 아동 장애·발달 정보(민감정보)를 위탁 처리할 때 추가로 필요한 조치가 있는지. 주민등록번호 미수집 방침으로 충분한지.
3. **수탁자 의무 조항(약관 제7조)**: 안전조치·재위탁 금지·목적 외 이용 금지·파기 의무 기재가 제26조 요건을 충족하는지. 별도 ‘위탁계약서’가 추가로 필요한지(약관·처리방침으로 갈음 가능한지).
4. **면책·책임 제한(약관 제3·9조)**: 지자체 점검 결과·바우처 환수·행정처분 등에 대한 면책 범위가 유효한지(특히 「약관규제법」상 무효 소지 점검).
5. **회원가입 전자동의(첨부 D)**: 가입 시 받는 ‘위탁 확약’ 전자 체크가 위탁계약 체결로 유효한지, 문구 보완점.
6. **이용요금 유료화 시**: 「전자상거래법」·「콘텐츠산업진흥법」상 청약철회·표시의무 등 추가 반영사항.
7. **만 14세 미만 아동**: 정보주체가 아동인 점과 관련해 추가 고려(법정대리인 동의 등)가 약관/방침에 충분히 반영됐는지.

## 4. 확정 필요 공란 (운영 정보 — 회사가 기재)

- 회사(운영자) 상호·대표자·사업자번호·주소·연락처
- 개인정보 보호책임자(성명·연락처)
- 데이터 저장 위치(국내/해외)·클라우드 수탁자명
- 보관기간(회원/아동정보 각각), 파기 절차
- 유료 전환 여부 및 요금·환불 정책
- 시행일·개정일
