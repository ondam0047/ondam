# 한글파일(.hwp) 출력 — 다음 단계 계획

> 작성: 2026-05-28
> 현재 상태: 일정표·기록지를 **.docx**로 다운로드 가능. 한글(아래아한글)에서 바로 열어 편집·저장 가능.
> 다음 목표: `.hwp` 또는 `.hwpx` **네이티브** 파일을 다운받게 하기.

---

## 왜 지금 .docx로 멈추는가

원장님이 주신 `시작가이드.md`에 이렇게 적혀 있어요:
> **한글파일(.hwp) 자동 생성**이 이 프로젝트에서 제일 까다롭습니다.
> 막히면, 우선 **PDF로 출력**되게 해서 전체 흐름을 먼저 완성하세요.
> 그 다음에 .hwp 생성을 따로 붙이면 마음이 편합니다.

샘플로 받은 `.hwp` 파일 4개는 모두 **HWP 5 바이너리(아래아한글 5.0+ 고유 형식)** 입니다.
이 형식을 Node.js에서 직접 생성하는 **검증된 라이브러리는 현재 시점(2026-05-28)에 없습니다.**
대신 다음 3가지 길이 있어요. 위에서 아래로 갈수록 결과는 완벽해지지만 작업이 많아져요.

---

## 길 A — `.docx` 그대로 사용 (현재 상태) ✅

- `[한글파일(.docx) 다운로드]` 버튼이 일정표·기록지 두 페이지에 이미 작동합니다.
- 한글(아래아한글) 2014 이후 버전은 `.docx`를 **메뉴 변경 없이 바로 열고 저장**합니다.
- 한 번 열어서 `다른 이름으로 저장 → 한글 문서(.hwp)`로 저장하면 끝.
- 양식 디자인은 우리 명세대로 (표·테두리·서비스 제공현황·서비스 비용 등) 똑같이 들어가요.
- **단점:** 한글에서 한 번 다시 열어야 함 (1초 추가됨).

> 원장님이 양식 서식을 가끔 손보는 편이라면 사실 이 방식이 가장 유연합니다.

---

## 길 B — `.hwpx` (한글 표준 XML 포맷) 생성 🟡

`.hwpx`는 .hwp의 **차세대 XML 기반** 포맷입니다.
- 구조: 사실상 **zip + xml** (워드의 .docx와 같은 패키지 구조)
- 한글 2010 이후 버전은 `.hwpx`를 **완전히 동등하게** 다룹니다.
- 한글 2014부터는 저장 시 .hwpx를 기본으로 권장하기도 함.

### 구현 방법
1. 원장님이 **한글에서 기존 양식을 한 번 `.hwpx`로 저장** 해주셔야 해요.
   (메뉴: `파일 → 다른 이름으로 저장 → 파일 형식: 한글 표준문서(.hwpx)`)
2. 그 .hwpx 파일을 `samples/`에 넣으면, 우리가 그걸 **템플릿**으로 삼아
   필요한 부분만 데이터로 치환합니다.
3. 치환 방법은 두 가지:
   - **단순 치환:** 양식 안에 `{{대상자}}`, `{{치료사}}` 같은 토큰을 직접 넣어두고 코드가 문자열 치환
   - **DOM 치환:** 표 셀을 XPath로 찾아 텍스트 노드만 교체 (더 정교, 더 까다로움)

### 필요한 작업
- [ ] 원장님이 한글에서 일정표·기록지 양식을 `.hwpx`로 저장해 보내주세요
- [ ] (개발) zip 압축/해제 (`jszip` 라이브러리 활용) — 이미 docx 생성에 쓰는 OOXML 노하우 그대로 적용 가능
- [ ] (개발) 위치별 치환 로직 작성

**예상 소요:** 1~2시간. **현실성: 매우 높음.**

---

## 길 C — `.hwp` 바이너리 직접 생성 🔴

- HWP 5 바이너리는 **OLE 복합 문서(CFBF)** 구조에 한컴 고유 record 포맷이 들어있어요.
- 오픈소스 파서(`@rhwp/core` 등)는 **읽기/렌더링만** 지원, 작성 기능 없음.
- 한컴오피스 SDK는 Windows 전용·유료.
- 이 길은 **시간 대비 효용이 가장 나쁩니다.** 추천하지 않습니다.

---

## 우리가 권장하는 진행 순서

1. **지금 단계 (Step 3 완료):** `.docx` 다운로드. 원장님 / 치료사 분들이 며칠 써보고 불편한 점 정리.
2. **Step 3.5:** 원장님이 양식을 `.hwpx`로 저장 → 토큰 삽입 → 우리가 자동 치환. (길 B)
3. **Step 4 (별도):** 엑셀 업로드를 DB와 연결해서 회기 기록을 영구 저장. (지금은 화면에서만 살아있음)
4. **Step 5:** 치료사 로그인.

## 참고: 현재 .docx 생성 코드 위치

- `app/api/schedule/docx/route.ts` — 일정표
- `app/api/record/docx/route.ts` — 기록지
- 둘 다 [`docx` 라이브러리](https://docx.js.org/) 사용. 표·테두리·셀병합·한글 폰트(맑은 고딕) 지원 확인됨.

`.hwpx` 단계로 넘어갈 때는 위 두 파일을 그대로 두고, 새 API 라우트
(`/api/schedule/hwpx`, `/api/record/hwpx`)를 추가하면 됩니다.
화면에는 다운로드 버튼을 두 개 두면 돼요 — `.docx`(범용)·`.hwpx`(한글 네이티브).
