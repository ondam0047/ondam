# 바로일지 UX 개선 — 회의 결론 & 구현 기록

> 디자인 전문가 + 사용자(치료사) 패널 회의 결과와 1차 구현 내역.
> 근거: 코드 검토(메뉴 9개·투어 9단계·도움말 11섹션·개념 중복) + 페르소나 인터뷰.

## 진단 (한 줄)
바로일지의 문제는 **기능 과잉이 아니라 계층 부재**. 같은 기능도 "월간 서류 공장"이라는
하나의 모델 아래 **빈도순 계층화 + 결과어 명명 + 첫 3분 최적화**하면, 신규 사용자가
90%를 무시하고도 성공한다.

## 북극성
> **"가입 후 첫 기록지까지 3분."** 모든 UX 결정은 이 숫자를 줄이는가로 판단.

## 핵심 마찰 지점 (근거)
1. 메뉴 9개가 전부 동등한 무게 — 매일 쓰는 것/가끔 쓰는 것 구분 없음.
2. 개념 중복: **일정표 vs 내 시간표**, **기록지 vs 일괄 다운로드**.
3. 투어가 데이터 0인 빈 화면을 9단계로 설명 → 기억 안 남음.
4. 첫 화면(대시보드)이 지표 위주 → 신규엔 빈 숫자판.
5. 도움말 11섹션 = 복잡도의 증상(제품이 스스로 설명 못 함).

## 처방 우선순위
| 순위 | 처방 | 크기 | 상태 |
|---|---|---|---|
| 1 | **첫 화면 시작 체크리스트** (아동→일정→기록지) | 小 | ✅ 구현 |
| 2 | **메뉴 2계층화** (핵심 작업 / 도구) | 小 | ✅ 구현 |
| 3 | 중복 제거: 일괄다운로드→기록지 안 버튼, 내시간표→일정표 주간보기 | 中 | 🟡 교차링크만 구현(병합 보류) |
| 4 | 이름을 결과어로 ("승인 빠진 거 찾기" 등) | 小 | ⬜ |
| 5 | 첫 로그인 데모 한 장 (seed 재활용) | 中 | ✅ 구현 |
| 6 | 홈을 행동 표면으로 (이번 달 마감 현황 + 다음 행동) | 中 | ✅ 구현 |
| 7 | (본 개편) 월 중심 워크스페이스 | 大 | ⬜ |

## 이번 1차 구현 (브랜치에 반영됨)

### ① 시작 가이드 체크리스트 — `app/(app)/dashboard/page.tsx`
- 신규 사용자가 **첫 기록지까지 가는 골든 패스**를 대시보드 최상단에 표시.
- 3단계: ☐ 첫 아동 등록 → ☐ 이번 달 일정 만들기 → ☐ 첫 기록지 받기.
- 각 단계는 실제 데이터로 완료 판정(`hasChild/hasSchedule/hasRecord`),
  **다음 할 단계만 강조 + CTA 버튼**, 완료 단계는 ✓·취소선·흐리게.
- **3단계 모두 끝나면 카드가 자동으로 사라짐**(온보딩 전용).
- 상단에 "처음이세요? 먼저 내 설정 확인" 안내 링크.
- 데이터: `loadMyStats`에 `everScheduleCount/everRecordCount`(평생 이력) 추가 →
  `hasChild/hasSchedule/hasRecord` 반환. (스키마 변경 없음)
- 기존 "아직 담당 아동이 없어요" 팁은 이 체크리스트로 대체.

### ② 메뉴 2계층화 — `app/(app)/Sidebar.tsx`
- `NAV_GROUPS`를 `{label?, items}[]` 구조로 변경, 빈도순 그룹:
  - (라벨 없음) 대시보드
  - **핵심 작업**: 일정표 · 기록지 · 내 아동
  - **도구**: 일괄 다운로드 · 승인내역 점검 · 내 시간표
  - (라벨 없음, 하단) 내 설정 · 도움말
- 라벨 그룹은 소제목(`nav-section`)으로, 무라벨 그룹은 구분선으로 렌더.
- 베타 운영자 메뉴는 "운영" 그룹으로 하단 직전 삽입.
- 메뉴 항목/이름은 그대로 유지(베타 사용자 혼란 방지) — 이름 변경(#4)은 별도 결정.

---

### ⑤ 첫 로그인 데모 한 장 — `app/api/record/sample/route.ts`
- 더미 데이터로 채운 **샘플 기록지 1장**을 즉석 생성·다운로드(`GET /api/record/sample`).
  센터가 고른 서식으로 생성 → "내 양식이 이렇게 채워지는구나"를 일하기 전에 체감.
- 시작 체크리스트에 "📄 샘플 기록지 먼저 보기" 링크.

### ⑥ 홈을 행동 표면으로 — `app/(app)/dashboard/page.tsx` `MonthFocusBanner`
- 온보딩 끝난(아동·일정·기록지 모두 있는) 사용자에겐 체크리스트 대신 **이번 달 행동 배너**:
  - 이번 달 일정 없음 → "일정 만들기"
  - 미작성 N명 → "이어서 작성"
  - 다 됨 → "일괄 다운로드" (🎉)
- 지표 카드 위에 "지금 할 일"을 먼저 노출.

### ③ 중복 정리(안전판) — `app/(app)/record/RecordClient.tsx`
- 기록지 화면 상단에 **"여러 명 한꺼번에 받기 →"** 교차 링크(→ 일괄 다운로드).
- 내 시간표↔일정표 **병합은 보류** — IA 변경이라 별도 결정(naming/주간보기 흡수).
