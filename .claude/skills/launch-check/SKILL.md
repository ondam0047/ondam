# 출시 체크리스트 관리 스킬 (LAUNCH.md)

「바로일지」의 출시 마스터 체크리스트 `LAUNCH.md`를 관리하는 스킬입니다.

## 언제 사용되나

다음 패턴의 사용자 발화에 자동 매칭:

- "출시까지 뭐 해야해" / "오늘 뭐 해야해" / "내일 뭐 할까"
- "체크리스트 보여줘" / "진행 상황" / "어디까지 됐어"
- "Week 1·2·3·4 뭐 남았어" / "이번 주 할 일"
- "[작업명] 완료" / "[작업명] 끝났어" / "[작업명] 했어"
- "출시 준비 어디까지" / "Go/No-Go 체크"
- `/launch` / `/출시` 슬래시 명령

## 표준 응답 절차

### Step 1 — LAUNCH.md 읽기

```
Read /home/user/ondam/LAUNCH.md
```

### Step 2 — 사용자 의도 판별

| 사용자 발화 | 처리 |
|---|---|
| **상태 조회** ("진행 상황", "어디까지") | 진행률·다음 액션·블로커 보고 |
| **시점별 조회** ("Week 1 뭐 남았어") | 해당 Week 미완료 항목 추출 |
| **완료 보고** ("X 끝났어") | 해당 항목 `[ ]` → `[x]` 수정 |
| **블로커 보고** ("X가 막혔어") | 해당 항목에 메모 추가 + 대안 제시 |
| **새 항목 추가** ("Y도 해야해") | 적절한 Week·카테고리에 신규 항목 추가 |
| **Go/No-Go 점검** ("출시 가능?") | 9개 출시 조건 매칭 보고 |

### Step 3 — 핵심 진행률 산정

LAUNCH.md 에서:
- 전체 `[ ]` vs `[x]` 비율
- Week 1·2~3·4·출시후 각각 진행률
- 🔴 P0 항목 미완료 수 (출시 막힘 후보)
- 출시 Go/No-Go 9개 매칭 수

### Step 4 — 다음 액션 추천 (3개)

미완료 항목 중 다음 우선순위로 정렬:

```
1순위: 🔴 P0 이번 주(Week 1) 항목
2순위: 🔴 P0 다음 시점(Week 2~3) 의존성 있는 항목
3순위: 🟠 P1 이번 주 항목
```

각 항목에 대해:
- 무엇을 하는 일인지 1줄
- 예상 소요 시간
- 비용 (해당 시)
- 막혔을 때 대안 (가능 시)

### Step 5 — 체크리스트 업데이트

사용자가 완료 보고하면 Edit 도구로 `[ ]` → `[x]` 변경.

여러 항목 동시 완료 시 한 번에 Edit (replace_all=false, 각각 unique context 확인).

## 응답 포맷 예시

### 진행 상황 조회 응답 (간결)

```markdown
## 📊 현재 진행 상황 (YYYY-MM-DD)

| 시점 | 진행률 | 미완료 P0 |
|---|---|---|
| Week 1 | 3/15 (20%) | 5개 |
| Week 2~3 | 0/22 | — |
| Week 4 | 0/14 | — |
| 출시 후 | 0/15 | — |

## 🎯 다음 3가지 액션

1. **사업자등록 완료** 🔴 P0 — 홈택스, 40분
2. **K-Startup 심화 상담 신청** 🔴 P0 — 30분, 자료 첨부 02_검토패키지.docx
3. **베타 사용자 모집글** 🔴 P0 — 네이버 카페 3곳, 30분

## 🚨 출시 Go/No-Go (9개 조건)
2/9 충족 → 출시까지 4~5주 예상
```

### 시점별 조회 응답

```markdown
## 🗓 Week 1 — 이번 주 미완료 P0 항목

### 사업자·세무
- [ ] 🔴 사업자등록 완료 (홈택스)
- [ ] 🔴 사업자등록증 PDF 보관

### 베타 준비
- [ ] 🔴 베타 사용자 모집 (시드 10~30명)
- [ ] 🔴 베타 소통 채널 (단톡방·디스코드)
- [ ] 🔴 베타 안내 페이지

→ 이번 주 P0 5개 남음. 1~2일 안에 가능한 항목부터 진행 권장.
```

### 완료 보고 처리 응답

```markdown
✅ "사업자등록 완료" 체크 반영했습니다.

## 다음으로 추천하는 1개
- 🔴 **사업자등록증 PDF 보관** — 다음 단계(결제대행사·통신판매업)에 필수.
  발급된 등록증을 PDF로 저장하고 `legal/business-cert.pdf` 보관 권장.

진행률 업데이트: Week 1 3/15 → 4/15 (27%)
```

### 새 항목 추가 응답

```markdown
LAUNCH.md 에 다음 항목 추가했습니다:

**Week 2~3 — 보안 자체 점검** 섹션 아래:
- [ ] 🟠 [추가한 항목 내용]

다른 시점이 더 적합하면 알려주세요.
```

## 주의사항

1. **LAUNCH.md 외 파일 수정 금지** — 다른 약관·코드 파일은 건드리지 않음
2. **완료 처리는 명시적 보고만** — "할 거야" / "곧 할게" 는 미완료로 유지
3. **P0 미완료가 있을 때 P2 추천 금지** — 우선순위 엄수
4. **출시 일정 거짓말 금지** — 진행률에 따라 정직하게 보고 ("출시까지 X주 더 필요")
5. **체크 반영 후 Bash로 커밋 X** — 사용자가 명시적으로 커밋 요청하지 않으면 변경만
